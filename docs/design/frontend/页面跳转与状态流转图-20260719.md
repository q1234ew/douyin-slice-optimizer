# 页面跳转与状态流转图

日期：2026-07-19

适用对象：

- 产品经理
- 设计师
- 前端开发
- 研发协作评审

## 文档目的

这份文档用于回答三个问题：

1. 用户在系统里的典型路径是什么
2. 页面之间应该如何跳转
3. 对象状态在页面之间如何流转

它是对 `功能逻辑与界面业务流程梳理` 的进一步落地，重点从“结构梳理”推进到“交互流转”。

## 一、系统总体流转图

```text
剪辑工作台

素材与节目
├── 已切短片
│   └── 批量排名结果
│       └── 候选审核
└── 完整节目
    ├── 节目理解
    ├── 候选召回
    ├── 候选评分
    └── 候选审核

候选审核
├── 通过
│   └── 发布准备
├── 需修整
│   └── 返回候选处理 / 保留当前队列
└── 暂缓
    └── 保留观察

发布准备
├── 包装配置
├── 字幕检查
├── 规格确认
└── 导出 / 发布

研究中心
├── 概览
├── 样本与标注
├── 评测与校准
├── 平台连接
└── 模型与环境
```

## 二、用户主路径

### 1. 高频生产路径

这是最核心的主流程：

```text
进入系统
→ 素材与节目
→ 选择节目 / 批次
→ 查看当前风险与下一步动作
→ 候选审核
→ 做出候选判断
→ 发布准备
→ 导出成片
```

这条链路要求页面设计始终围绕三个问题：

- 当前对象是谁
- 当前判断是什么
- 下一步去哪里

### 2. 异常排查路径

当用户对结果不放心时，路径会切换为：

```text
工作台发现异常
→ 研究中心概览
→ 进入评测与校准 / 平台连接 / 模型与环境
→ 确认问题来源
→ 回到剪辑工作台继续处理
```

这条链路要求研究中心页面始终围绕两个问题：

- 当前最重要的问题是什么
- 这个问题是否阻塞主链路

## 三、页面跳转关系

## 1. 顶层模式切换

### `剪辑工作台 ↔ 研究中心`

跳转目的：

- `剪辑工作台 → 研究中心`
  - 需要解释结果
  - 需要确认系统状态
  - 需要排查排序、校准、授权、预算或环境异常
- `研究中心 → 剪辑工作台`
  - 异常已确认
  - 当前问题不再阻塞主链路
  - 需要继续处理节目和候选

设计要求：

- 第一层导航必须是模式切换，不承载过多说明
- 用户应该始终清楚自己当前处于“生产”还是“研究”

## 2. 剪辑工作台内部跳转

### `素材与节目 → 候选审核`

触发条件：

- 节目已生成候选
- 已切短片完成统一排序
- 用户开始人工判断

传递信息：

- 当前节目 / 批次 identity
- 候选队列
- 当前质量结论
- 风险摘要

不应丢失的信息：

- 来源对象
- 当前处理阶段
- 是否存在待复核风险

### `候选审核 → 发布准备`

触发条件：

- 当前候选确认通过
- 或进入可导出状态

传递信息：

- 当前候选 identity
- 当前通过状态
- 标题建议
- 风险与修整结果

不应丢失的信息：

- 通过理由
- 当前字幕 / 包装风险
- 导出优先级

### `发布准备 → 导出`

触发条件：

- checklist 满足
- 包装与字幕已达可交付状态

传递信息：

- 标题 / 封面版本
- 成片规格
- 当前候选来源
- 导出版本

## 3. 研究中心内部跳转

### `概览 → 样本与标注`

适用场景：

- 需要补样本
- 需要校正标注质量
- 需要理解数据基础是否稳定

### `概览 → 评测与校准`

适用场景：

- 当前排序结果不稳
- benchmark 漂移
- 校准误差上升
- 出现严重错判

### `概览 → 平台连接 / 模型与环境`

适用场景：

- 指标回流异常
- 授权失效
- Provider 不可用
- 预算或环境限制影响主链路

设计要求：

- 研究二层导航是任务切换，不是“空 tab”
- 每次跳转都应带着“当前异常上下文”

## 四、对象状态流转

## 1. 节目 / 批次状态流转

```text
待处理
→ 已转写
→ 已生成候选
→ 已评分
→ 待复核
→ 已完成
```

异常分支：

```text
任意阶段
→ 部分异常
→ 回到当前阶段修复
→ 继续主流程
```

页面映射：

- `素材与节目` 承担全部状态可视化
- `研究中心` 只在异常来源解释时介入

## 2. 候选状态流转

```text
待审核
→ 建议放行
→ 确认通过
→ 进入发布准备
→ 已导出
```

分支状态：

```text
待审核
→ 需修整
→ 返回当前候选继续处理

待审核
→ 继续观察
→ 保留在候选队列
```

页面映射：

- `候选审核` 是状态判断中心
- `发布准备` 承接通过后的后续状态

## 3. 研究状态流转

```text
稳定在线
→ 持续观察
→ 待处理
→ 受控放行 / 挂起中 / 已关闭
→ 恢复稳定
```

页面映射：

- `研究中心概览` 展示当前综合状态
- `评测与校准` 展示方法与结果风险
- `平台连接与模型` 展示支撑与连接风险

## 五、各页面的入口、出口和主动作

## 1. 素材与节目

入口：

- 默认进入
- 从其他工作台页返回

主动作：

- 选择节目 / 批次
- 查看风险结论
- 开始复核
- 进入候选审核

出口：

- 候选审核
- 研究中心概览

## 2. 候选审核

入口：

- 素材与节目
- 研究中心返回后重新确认候选

主动作：

- 通过
- 需修整
- 继续观察
- 进入发布准备

出口：

- 发布准备
- 回到候选队列
- 研究中心

## 3. 发布准备

入口：

- 候选审核通过后进入

主动作：

- 调整标题
- 选择封面
- 检查字幕
- 通过 checklist
- 导出

出口：

- 导出结果
- 回到候选审核重新修整

## 4. 研究中心概览

入口：

- 工作台发现异常
- 主动查看系统健康

主动作：

- 看当前关键异常
- 进入对应研究子页

出口：

- 样本与标注
- 评测与校准
- 平台连接
- 模型与环境
- 回到剪辑工作台

## 六、界面设计必须支持的使用逻辑

为了让页面真正好用，而不是只是信息完整，设计上必须保证：

### 1. 顶层跳转永远先确认“我在哪”

用户要先知道：

- 我现在是在 `剪辑工作台`
- 还是在 `研究中心`

然后才是：

- 我在这个模式里的哪个阶段

### 2. 每个页面都要先回答“现在该干嘛”

不能先展示一大堆说明。

应优先展示：

- 当前对象
- 当前状态
- 当前主动作

### 3. 状态必须能够驱动跳转

例如：

- `待复核` 状态对应进入候选审核
- `确认通过` 状态对应进入发布准备
- `待处理` 状态对应进入研究中心对应子页

### 4. 页面之间必须携带上下文

不能出现用户跳页后还要重新理解：

- 当前处理的是哪个节目
- 当前候选是哪条
- 当前异常对应哪一个 batch

## 七、当前最值得继续补的逻辑空缺

如果后续还要继续收敛产品逻辑，建议优先补这几个点：

### 1. 推荐模拟的归属

当前它更像：

- 候选判断依据

而不是：

- 独立主页面

建议正式归并到候选审核详情中。

### 2. 候选修整回流逻辑

当前需要进一步明确：

- `需修整` 是返回原候选重新编辑
- 还是产生新版本
- 修整后是否重新进入排序

### 3. 发布后的回流入口

当前主设计稿已覆盖到导出前，但后续还应明确：

- 发布后结果如何回到研究中心
- 是否需要单独“结果回流”视图

## 八、结论

页面跳转与状态流转的核心原则可以概括为：

先用顶层模式告诉用户“你现在在哪个工作域”，再用阶段导航告诉用户“你正在做哪一步”，最后用页面主区域告诉用户“现在该做什么”。

只要这个逻辑统一，设计、产品和前端实现都会稳定很多。
