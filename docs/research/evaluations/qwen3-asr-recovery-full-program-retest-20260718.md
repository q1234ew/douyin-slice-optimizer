# Qwen3-ASR 漏段恢复策略完整节目复测

复测日期：2026-07-18  
Benchmark：`dso-qwen3-asr-recovery-20260718-r2`

## 结论

本轮恢复策略完成了 7506.73 秒完整节目的冻结复测。60 秒主块、1 秒重叠、5 秒低能量边界搜索、空 context 和异常块 30 秒缩块重试能够稳定跑完整段节目；此前 180 秒配置静默漏掉的“这首歌有着非常蜿蜒的旋律”和“歌手们今晚的命运”均重新命中，音频块覆盖率为 100%，没有未解决块或 context 回显，端到端 RTF 为 `0.050345`。

仅看本轮单节目，该结果还不足以让 Qwen3-ASR 替代 Whisper：10 个可见字幕锚点仍命中 7 个，与旧固定 60 秒基线持平；2 个触发自动恢复的主块中，1 个只恢复出单字“啊”，属于低信息假恢复，另 1 个是缺少逐字稿 Gold 的歌词扩展。本轮评测结束时模型状态因此保持 `shadow`。

状态更新（2026-07-18）：第二个独立完整节目评测完成后，两个节目冻结锚点合计为 Qwen `15/22`、Whisper small `7/22`、base `4/22`。根据后续用户决策，生产路由已调整为 Qwen3-ASR 主用、Whisper 自动回退；本报告保留上述单节目审慎结论作为历史评测记录，不代表当前部署状态。

## 恢复块人工复核

### Chunk 9：510.569–569.959 秒

- 初始请求：空结果，耗时 0.311 秒。
- 3 个缩块重试：每个都只返回 1 个字符。
- 最终选择文本：“啊”。
- 固定 60 秒基线：空文本。
- 判断：`false_recovery_low_information`。

这个案例说明“未解决块为 0”只能证明程序找到了可接受格式的输出，不能证明恢复了有意义口播。后续应把多个子块都只返回单字语气词的情况标为 `no_speech` 或 `unresolved_low_information`，不能计作成功恢复。

### Chunk 26：1482.433–1541.620 秒

- 初始请求：8 个字符，耗时 19.762 秒，触发慢且稀疏恢复。
- 缩块后：得到 26 个字符的歌词组合。
- 固定 60 秒基线：“你我皆是凡人躯嘿嘿一世贪欲始有起”。
- 新结果：“一出好戏做不透你我皆是凡人躯嘿嘿嘿一朝爱恨来一世贪欲一世有情”。
- 判断：`plausible_lyrics_expansion_not_key_speech`。

新文本看起来更完整，但没有逐字稿 Gold，不能把字符增加直接算成准确率提升。

另有一个未触发重试的慢块（3356.009–3414.722 秒，19.771 秒、52 字）；与基线窗口人工比较未发现可见截断，新结果还保留了副歌后的感谢口播，因此本轮没有仅因耗时长而强制重试。

## 下一步

1. 在恢复选择器中拒绝“所有子块均只有单字语气词”的低信息组合，分别记录 `no_speech` 与 `unresolved_low_information`。
2. 将 RMS 活跃音频判断升级为 VAD/语音占比联合门控，避免把音乐尾音或噪声当成应恢复口播。
3. 服务端记录生成 token 数、结束原因和对齐耗时，继续定位约 20 秒慢解码平台期。
4. 建立至少 3–5 个完整节目、覆盖主持口播、歌曲、专名、多人和中英混说的人工逐字稿 Gold；同一冻结集并排评测 Qwen 与 Whisper 的 CER/WER、漏段率和成本。
5. 在 ASR Gold 门禁通过后，再用人工切片 Gold 验证候选 Recall@K、边界误差和最终排序 NDCG@K，不能用转写字符数替代业务收益。
