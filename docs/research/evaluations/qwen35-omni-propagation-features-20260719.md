# Qwen3.5-Omni 传播特征抽取与结果映射试验

- 日期：2026-07-19
- 状态：`research_only`
- 冻结结果：`benchmarks/dso-omni-propagation-features-v2-20260719-r1.json`

## 目标

本轮将完整短片研究拆成两个互相隔离的阶段：

1. 完整音视频只交给 Omni 抽取可观察的音画、时序和叙事特征。
2. Provider 全部返回后，再在本地连接真实平台结果字段。

Omni 请求不包含账号、标题、历史标签、互动数或既有策略结果，也不要求模型输出传播分、流量分或爆款概率。实验不修改生产排序、人工 Gold、导出或发布状态。

## 工程实现

- 新增独立 profile `qwen35_omni_propagation_features`，固定 `qwen3.5-omni-plus-2026-03-15`。
- 新增 `propagation_feature_outcome_dataset.v1`，结构化保存内容形态、钩子、音频、视觉、叙事、时间线、限制、置信度和弃权。
- 新增 `scripts/run_bailian_propagation_features.py`，支持 1-100 条、零网络预检、显式批次预算、缓存复用和逐请求台账。
- 分享率、关注转化率和观看质量缺少必要字段时保持 `unavailable_*`，不以 0 或互动数代替。
- 错误台账只记录校验器生成的字段诊断，不保存模型原文、媒体、提示词或密钥。

## Schema 稳定性

v1 首批 10 条只有 2 条通过，字段级诊断确认主要是两类机械问题：音画事件窗口自然重叠被误判为乱序，以及 `limitations` 类型或条数不稳定。修复没有改写模型语义，只做确定性结构归一化：

- 时间线按起点排序，允许音画窗口重叠，最多保留 6 项。
- `limitations` 单字符串包装为单项数组，最多保留 5 项。
- prompt v2 明确 `limitations` 必须为字符串数组。

v2 的 3 条 smoke 为 3/3 成功；冻结 10 条全量为 10/10 成功。时间线、音频证据和视觉证据覆盖均为 100%，无弃权。

## 成本与延迟

冻结全量复用了 3 条 smoke 缓存，新增 7 次网络请求，usage 估算为 `0.733441 CNY`。成功请求延迟中位数/P95 为 `19.348/23.188s`。随后生成关联摘要时 10 条全部命中缓存，新增请求和费用均为 0。包含 v1、字段诊断、归一化 smoke、v2 smoke 和冻结全量在内，schema 稳定化过程累计 26 次网络请求，usage 估算 `2.509202 CNY`。

本轮执行使用显式 `50 CNY` 批次硬上限作为异常停止线，不代表建议花满，也不把赠送 Token 视为零成本。日常环境没有持久开放 `full_media`。

## 结果字段可用性

| 目标 | 覆盖 | 当前口径 |
| --- | ---: | --- |
| 可见互动热度 | 10/10 | 账号内校正后的点赞、评论、收藏、分享代理，不是播放量 |
| 分享率 | 0/10 | 缺少播放/曝光分母 |
| 关注转化率 | 0/10 | 缺少播放/曝光分母和新增关注 |
| 观看质量 | 0/10 | 缺少 5 秒留存、平均观看比例和完播率 |

因此，本轮只能验证“完整音视频特征与可见互动热度代理的关联”，不能声称已经提炼出分享、关注或观看质量机制。

## 探索性发现

5 条 high 与 5 条 low 的配对诊断中，high 侧观察到较高的低上下文依赖、高字幕密度和中等切镜密度；low 侧更多出现高上下文依赖和低切镜密度。样本量只有每组 5 条，且样本是按已知互动差异富集选择，结论仅用于形成下一轮假设，不能视为因果机制或排序权重。

模型原生 `traffic_potential_score` 继续拒绝进入 ranker。下一轮只验证事实特征的独立增量。
